# ビジョン

## なぜ作るのか

### 解決したい課題

| 課題 | 現状 | 影響 |
|------|------|------|
| 過去のやり取りを忘れる | 記憶頼み | 同じ対応を繰り返す |
| 会話がアプリ内だけ | WhatsAppに散在 | 引き継ぎ困難 |
| ファイルが埋もれる | 整理されていない | 証跡が探せない |
| 提出物管理が曖昧 | 記憶頼み | 催促忘れ |
| 通話内容が残らない | 記録なし | 言った言わない問題 |

### 理想の状態

- 「前にどんなやり取りしたっけ？」→ すぐ検索できる
- 「この資料の内容は？」→ AIが回答してくれる
- 「提出期限いつだっけ？」→ 自動でリマインド
- 引き継ぎ → 履歴を渡すだけ

## 最終ゴール

**請負者とのすべてのやり取りが、検索可能な形で蓄積され、AIが補助してくれる状態**

### 具体的には

1. すべての会話・ファイル・通話が自動保存される
2. 過去のやり取りを自然言語で検索できる
3. 保存した資料をAIに質問できる
4. 忘れそうなことをAIがリマインドしてくれる
5. 複数の請負者を安全に分離管理できる
6. **AIは用途・コストに応じて柔軟に切り替え可能**

## 部屋構造のコンセプト
```
Workspace（最上位の部屋）
│
├── 完全分離（Workspace A と B は相互に見えない）
│
├── Workspace A
│   ├── Room 1（トピック：技術相談）
│   ├── Room 2（トピック：事務連絡）
│   └── Room 6（統合：AI + 依頼者用）
│       └── Room 1, 2 の情報を集約
│
└── Workspace B
    ├── Room 3
    ├── Room 4
    └── Room 5
```

### ルール
- Workspace間：完全分離、情報共有なし
- Room間（同一Workspace内）：情報共有可能（設定次第）
- 統合Room：複数Roomの情報を集めてAI分析・通知

## 対象ユーザー

### 依頼者（あなた）
- 複数の請負者とやり取りする
- 過去のやり取りを忘れがち
- コードは読めないが、ツールは使える

### 請負者
- 依頼者とチャット・通話・ファイル共有する
- 新しいツールへの移行は可能
- ITリテラシーは様々

## 成功の定義

| 指標 | 目標 |
|------|------|
| 「忘れてた」の削減 | 月1回以下 |
| 過去やり取り検索 | 30秒以内に見つかる |
| 引き継ぎ時間 | 1時間以内 |
| 請負者の移行 | 1週間で完了 |
