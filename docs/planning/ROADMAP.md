## Phase 1: MVP

### 目的
最小限の機能で動くものを作り、1社で試験運用する

### 期間
4週間

### 機能
| 機能 | 内容 |
|------|------|
| メッセージ保存 | 全テキストをDBに保存 |
| ファイル保存 | 写真・動画・音声をローカル保存 |
| Workspace分離 | 請負者ごとに完全分離 |
| Room管理 | トピック・メンバーごとの部屋 |
| /summary | 会話要約 |
| /search | 過去メッセージ検索 |
| 通知 | 依頼者専用チャンネルへ通知 |
| AI切替 | 機能ごとにAIプロバイダー設定可能 |

### 環境
- ローカル（macOS / Windows）
- uv + ruff + pre-commit
- SQLite
- ローカルファイル保存

### 完了条件
- [ ] 5つのWorkspace（請負者）を管理できる
- [ ] メッセージ・ファイルが保存される
- [ ] /summary, /search が動作する
- [ ] AI設定を変更できる
- [ ] 1社で1週間試験運用

## Phase 2: 拡張機能

### 目的
通話録音とGoogle Drive連携を追加

### 期間
3週間

### 機能
| 機能 | 内容 |
|------|------|
| 通話録音 | 自動録音（on/off切替可） |
| 文字起こし | Whisper APIで音声→テキスト |
| Google Drive | ファイルをDriveに保存 |
| フォルダ指定 | 手動で保存先を指定 |
| /docs | 資料一覧表示 |
| /remind | リマインダー登録 |

## Phase 3: RAG + クラウド移行

### 目的
資料を読んでAIが回答できるようにし、常時稼働させる

### 期間
3週間

### 機能
| 機能 | 内容 |
|------|------|
| テキスト抽出 | PDF・Wordから内容抽出 |
| ベクトル検索 | 資料を意味で検索 |
| /ask | 資料を基にAI回答 |
| クラウド移行 | Fly.ioで常時稼働 |
| PostgreSQL | クラウドDB |
| Docker | コンテナ化 |
