# リファクタリング

$ARGUMENTS

## リファクタリング手順

1. **現状分析**
   - 対象コードの問題点を特定
   - 改善の方向性を決定

2. **テスト確認**
   - 既存テストが十分にあることを確認
   - 不足していればテストを追加

3. **リファクタリング実行**
   以下の観点で改善：
   - 重複コードの排除（DRY原則）
   - 関数・クラスの責任分離（単一責任原則）
   - 命名の改善
   - 複雑度の低減
   - 型ヒントの追加・修正

4. **テスト通過確認**
   - `uv run pytest tests/ -v`
   - 動作が変わっていないことを確認

5. **リント・型チェック**
   - `uv run ruff check src/`
   - `uv run pyright src/`
