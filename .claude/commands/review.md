# コードレビュー

$ARGUMENTS

## レビュー実行

以下の専門エージェントに並列でレビューを依頼します。

1. **python-expert** に依頼：Pythonのベストプラクティス観点でレビュー
2. **security-reviewer** に依頼：セキュリティ観点でレビュー
3. **code-reviewer** に依頼：可読性・保守性観点でレビュー
4. **qa-reviewer** に依頼：テスト品質観点でレビュー
5. **discord-expert** に依頼：Discord API観点でレビュー（該当する場合）

## レビュー結果の統合

各エージェントのレビュー結果を以下の形式で統合してください。

### 🔴 Critical（必須修正）
- 修正しないとバグやセキュリティ問題が発生するもの

### 🟡 Warning（推奨修正）
- 修正した方が良いが、動作には影響しないもの

### 🟢 Suggestion（提案）
- より良くするための提案

### ✅ Good Points（良い点）
- 良い実装パターンや工夫
