# テスト作成

$ARGUMENTS

## テスト作成手順

1. **対象コードの理解**
   - 対象ファイルの機能を把握
   - 入出力、副作用を確認

2. **テストケース設計**
   docs/TEST_PLAN.md を参照し、以下の観点でテストケースを設計：
   - 正常系：期待通りの入力で期待通りの出力
   - 異常系：不正な入力でのエラーハンドリング
   - 境界値：限界値付近の動作

3. **テスト実装**
   - tests/ にテストファイルを作成
   - pytest形式で記述
   - fixtureを適切に使用

4. **テスト実行**
   - `uv run pytest tests/ -v` で実行

5. **カバレッジ確認**
   - `uv run pytest tests/ --cov=src --cov-report=html`
   - 70%以上を目標
